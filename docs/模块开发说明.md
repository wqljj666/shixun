# 模块开发说明

## 本次开发目的

本次开发的目标是完成“线上购药系统”课程项目的主要功能模块，并整理为可提交、可运行、可从 GitHub 下载 ZIP 的完整工程。项目包含两个传统业务模块和一个借助大模型实现的智能化应用模块。

## 开发工具

- JDK 17
- Maven
- Spring Boot
- H2 Database
- Thymeleaf
- Bootstrap 5
- Bootstrap Icons
- VS Code / IntelliJ IDEA 均可运行
- Codex 辅助开发

## 使用的大模型工具

开发过程中使用 Codex 作为大模型辅助编码工具，用于生成代码结构、补充方法注释、优化页面样式、整理文档和辅助检查项目交付项。

项目运行时的智能药师模块支持 OpenAI Chat Completions 兼容接口，默认配置为 DeepSeek 接口地址。未配置 API Key 时会自动使用本地模拟回复。

## 传统功能模块一：药品信息模块

药品信息模块用于展示线上药房中的药品基础信息。

完成的功能：

- 药品分类展示
- 药品列表展示
- 药品关键词搜索
- 药品分类筛选
- 药品详情查看
- 药品库存展示
- OTC / 处方药标签展示

主要页面：

- `/medicines`
- `/medicines/{id}`

## 传统功能模块二：购物车与订单模块

购物车与订单模块用于模拟医药电商中的选购、结算和支付流程。

完成的功能：

- 加入购物车
- 查看购物车
- 修改购物车数量
- 删除购物车商品
- 创建订单
- 查看订单列表
- 查看订单详情
- 取消待支付订单
- 模拟支付

主要页面：

- `/cart`
- `/orders/confirm`
- `/orders`
- `/orders/{id}`

## 智能化功能模块：智能药师咨询模块

智能药师咨询模块用于满足课程中“一个借助大模型实现智能化应用的功能模块”的要求。

完成的功能：

- 用户输入症状、用药问题或健康咨询内容
- 后端生成 AI 药师回复
- 支持配置大模型 API
- 未配置 API Key 时使用本地模拟回复
- 高风险关键词识别
- 高风险问题提示转人工药师
- 保存咨询记录
- 查看历史咨询记录
- 所有 AI 回复包含免责声明

高风险关键词包括：

```text
孕妇、儿童、婴儿、老人、青霉素过敏、过敏史、处方药、抗生素、剧烈疼痛、呼吸困难、高烧、慢病、多药同服、肝肾功能异常
```

主要页面：

- `/ai-consult`
- `/ai-consult/history`

## 大模型在编码过程中的作用

大模型主要参与了以下工作：

- 根据课程要求拆分模块和目录结构；
- 生成 Spring Boot 实体、Repository、Service、Controller 基础代码；
- 辅助设计业务规则；
- 辅助生成 Thymeleaf 页面；
- 辅助统一 Bootstrap 风格和 CSS；
- 辅助补充 README 和 docs 文档。

## 大模型如何辅助生成方法注释

在最终整理阶段，大模型根据每个 Service 的职责和业务规则，为核心类和方法补充 JavaDoc 注释。注释说明了类职责、方法功能、入参、返回值、关键业务规则和异常处理逻辑。

补充注释的核心类包括：

- `MedicineService`
- `CartService`
- `OrderService`
- `PaymentService`
- `AiConsultationService`
- `LargeModelService`
- `RiskDetectService`

## 大模型如何辅助自动补全

大模型根据已有命名和项目结构自动补全：

- 实体字段和 getter / setter；
- Repository 查询方法；
- Service 业务方法；
- Controller 路由方法；
- Thymeleaf 页面绑定表达式；
- CSS 类名和响应式样式。

## 大模型如何引导生成代码

开发时先由人工给出课程要求，再由大模型根据要求生成初版代码。随后人工通过运行、阅读和功能验证检查代码是否满足业务规则，再对不合适的实现进行修改。例如智能药师模块中，高风险咨询不会直接给出具体用药结论，而是提示转人工药师或医生确认。

## 如何运行项目

```bash
cd online-pharmacy-system
mvn spring-boot:run
```

如需配置大模型 API Key，可在 PowerShell 中临时设置：

```powershell
$env:LLM_API_KEY="你的API_KEY"
$env:LLM_MODEL="deepseek-v4-flash"
mvn spring-boot:run
```

## 如何测试项目

可按以下顺序测试：

1. 访问首页：`http://localhost:8080`
2. 访问药品列表：`http://localhost:8080/medicines`
3. 搜索药品并进入详情页。
4. 将非处方药加入购物车。
5. 进入购物车修改数量并结算。
6. 创建订单后查看订单详情。
7. 模拟支付或取消待支付订单。
8. 进入智能药师页面提交普通咨询。
9. 提交包含“儿童”“高烧”“青霉素过敏”等关键词的高风险咨询。
10. 进入咨询记录页检查记录保存情况。
